# Safe GitHub Achievements Strategy

Objectif: progresser sur les achievements sans comportement spammy ou à risque.

## Principes
- Prioriser des contributions utiles, même petites.
- Éviter les rafales massives de PR/issues/discussions.
- Garder un historique propre (messages clairs, branches supprimées).
- Ne pas manipuler artificiellement stars/sponsors/comptes.

## Rythme recommandé
- 1 contribution/jour maximum (PR ou discussion ou issue utile).
- 5 à 15 minutes entre étapes quand possible (création -> ajustement -> merge).
- Arrêt immédiat si aucun changement utile n’est identifié.

## Checklist quotidienne
1. Vérifier qu’il n’y a pas de PR/issue ouverte à traiter.
2. Faire une amélioration minimale mais réelle (doc, test, script, typo, clarif).
3. Ouvrir une PR propre avec description concise.
4. Merge en squash et suppression de branche.
5. Vérifier que le repo reste propre (0 PR ouverte non voulue).

## Sécurité / conformité
- Respect des ToS GitHub.
- Aucune automatisation agressive.
- Aucune action sur des repos tiers sans intention réelle de contribution.
